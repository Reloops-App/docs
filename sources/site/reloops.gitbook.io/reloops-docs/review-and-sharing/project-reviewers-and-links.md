### 

Project reviewers and links

Project review ownership is visible from the Reviewers and Links tabs.

App route: `/workspace/:workspaceId/projects/:projectId`

![](https://reloops.gitbook.io/reloops-docs/~gitbook/image?url=https%3A%2F%2Fcontent.gitbook.com%2Fcontent%2FqpxgNvXAcPEWnIZeWuLE%2Fblobs%2FK1l1y0QYYePPm8Mx3CHU%2Fproject%2520reviewers%2520tab%2520guest%2520reviewer.png&width=768&dpr=3&quality=100&sign=4d6e99c0&sv=2)

Project reviewers tab showing a guest reviewer and assigned asset

Use the Links tab to check which asset links are active and whether they expire.

![](https://reloops.gitbook.io/reloops-docs/~gitbook/image?url=https%3A%2F%2Fcontent.gitbook.com%2Fcontent%2FqpxgNvXAcPEWnIZeWuLE%2Fblobs%2Fw8XyAAQXYiM2MaRKqQgO%2Fproject%2520share%2520links%2520tab%2520active%2520link.png&width=768&dpr=3&quality=100&sign=68c486cc&sv=2)

Project links tab showing an active asset share link

### 

When to check these tabs

- A reviewer says they cannot access an asset.
 
- A project has several external stakeholders.
 
- A review window is finished and old links should be cleaned up.
