# Assets overview

## What are assets in Reloops

Assets are the files you manage and review inside Reloops. Every file you upload into a project becomes an asset.

Assets are the core of your work. They are where feedback happens, decisions are made and progress is tracked.

\*\*\*

## Where assets live

Assets always live inside a project. You cannot create an asset outside a project.

The structure looks like this:

\* A workspace contains projects
\* A project contains assets

This keeps your files organized and tied to the correct work.

\*\*\*

## What you can do with an asset

Each asset has its own space where you can manage and review it.

!\[Asset viewer interface showing comments and annotations\](/files/9f959c0ab5ac4fcb270f96da397590b183ed840e)

With an asset, you can:

\* preview the file
\* leave comments
\* add annotations
\* assign it to a teammate
\* update its status
\* approve it when ready
\* download the original file
\* view its history and details

All feedback and activity stay connected to the asset itself.

\*\*\*

## Asset statuses

Every asset can move through simple statuses that reflect its progress.

The available statuses are:

\* \*\*No status\*\*
\* \*\*Needs review\*\*
\* \*\*In Review\*\*
\* \*\*Approved\*\*

Updating an asset’s status helps your team understand what needs attention and what is complete.

Statuses are visible on the asset card and inside the Kanban view.

\*\*\*

## Asset views inside a project

Assets appear in two main views inside a project.

### Files view

The Files view shows assets as cards. This view is useful for browsing, searching, filtering and taking quick actions.

!\[Files view showing asset cards\](/files/b290e7139963e4282546780a946b2c43f8f0111e)

### Kanban view

The Kanban view organizes assets into columns based on their status. This view is helpful for tracking progress across many assets at once.

Both views always stay in sync.

\*\*\*

## Collaboration on assets

Assets are designed for collaboration.

You can:

\* discuss changes using comments
\* point to exact areas using annotations
\* mark feedback as complete
\* invite reviewers to participate

This keeps conversations clear and actionable.

\*\*\*

## What happens after upload

After you upload a file:

\* it becomes an asset
\* it appears in the Files view
\* it can be reviewed and tracked
\* it moves through statuses as work progresses

From there, you can manage the entire review process from the asset itself.

\*\*\*
